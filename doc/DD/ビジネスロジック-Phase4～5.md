# ■ 1. 目標管理

---

## ■ 1.1 目標設定

### ■ 概要

ユーザーの月ごとの学習時間目標を登録する

---

### ■ 処理フロー

1. ログインユーザーIDを取得
2. リクエストの month, targetMinutes を取得
3. 同一ユーザー・同一monthの目標存在チェック
4. 存在する場合はエラー（409）を返却
5. Goalエンティティを作成
6. 保存
7. レスポンス返却（id）

---

### ■ 入力

* month（YYYY-MM）
* targetMinutes

---

### ■ 出力

* 目標ID

---

### ■ バリデーション

| 項目            | 内容           |
| ------------- | ------------ |
| month         | 必須 / YYYY-MM |
| targetMinutes | 必須 / 1〜20000 |

---

### ■ エラー

| 条件    | 内容         |
| ----- | ---------- |
| 重複（月） | 409（重複エラー） |

---

---

## ■ 1.2 目標一覧取得

### ■ 概要

ユーザーの目標一覧と達成状況を取得する

---

### ■ 処理フロー

1. ユーザーIDを取得

2. 目標一覧を取得（user_idで絞る）

   * month DESC（新しい順）

3. 各目標について以下を実施

   4. monthから対象期間を算出

      * 開始日：YYYY-MM-01
      * 終了日：月末（YearMonthで算出）

   5. StudyRecordから対象期間の学習時間を集計（SUM）

      * user_id一致
      * date BETWEEN 開始日 AND 終了日

   6. achievedMinutesとして設定

7. DTOに変換

8. レスポンス返却

---

### ■ 出力

* 目標一覧（id, month, targetMinutes, achievedMinutes）

---

### ■ 補足

* 集計処理はDB側で実施（SUM）
* monthはServiceでLocalDateに変換して使用

---

# ■ 2. GitHub連携

---

## ■ 2.1 コミット数取得

### ■ 概要

指定期間のGitHubコミット数を取得する

---

### ■ 処理フロー

1. ユーザーID取得

2. usersテーブルから githubUsername を取得

3. githubUsernameが未設定の場合

   * totalCommits = 0 を返却

4. GitHub API呼び出し準備

   * WebClient使用
   * PATをAuthorizationヘッダに設定

5. リポジトリ一覧取得

   * GET /users/{username}/repos

6. 各リポジトリについて以下を実施

   7. コミット一覧取得

      * GET /repos/{owner}/{repo}/commits
      * クエリ：since / until

   8. 最大100件まで取得

   9. 件数をカウント

10. 全リポジトリのコミット数を合計

11. レスポンス返却

---

### ■ 出力

* totalCommits

---

### ■ エラー

| 条件           | 内容     |
| ------------ | ------ |
| パラメータ不正      | 400エラー |
| GitHub API失敗 | 500エラー |

---

### ■ 補足

* publicリポジトリを対象とする
* 1リポジトリあたり最大100件まで取得（制限あり）
* API呼び出しは逐次実行（並列化しない）

---

# ■ 3. ストリーク

---

## ■ 3.1 連続学習日数取得

### ■ 概要

現在の連続学習日数（ストリーク）を取得する

---

### ■ 処理フロー

1. ユーザーID取得

2. 学習記録から学習日一覧を取得

   * DISTINCT date
   * user_idで絞る

3. 日付を降順でソート

4. 現在日を取得（LocalDate.now()）

5. ストリーク計算

   6. currentDate = 今日

   7. ループ処理

      * currentDateが学習日一覧に存在する場合

        * streak++
        * currentDateを1日減算
      * 存在しない場合

        * ループ終了

8. レスポンス返却

---

### ■ 出力

* currentStreak

---

### ■ 補足

* 同日に1件以上の記録があれば1日とカウント
* DBではなくアプリ側で計算

# ■ 4. エクスポート

---

## ■ 4.1 CSVエクスポート

### ■ 概要

指定期間の学習記録をCSV形式で出力する

---

### ■ 処理フロー

1. ユーザーID取得

2. パラメータ from / to を取得

3. 学習記録を取得

   * user_id一致
   * date BETWEEN from AND to

4. date昇順でソート

5. CSV文字列生成

   6. ヘッダ行追加

      ```
      date,subject,minutes,memo
      ```

   7. 各レコードを1行ずつ追加

8. レスポンス返却（text/csv）

---

### ■ 入力

| 項目   | 内容  |
| ---- | --- |
| from | 必須  |
| to   | 必須  |

---

### ■ 出力

* CSVデータ

---

### ■ エラー

| 条件      | 内容     |
| ------- | ------ |
| パラメータ不正 | 400エラー |

---

---

# ■ 共通補足

---

## ■ データ取得範囲

* すべてユーザー単位で取得
* user_idで必ず絞る

---

## ■ パフォーマンス

* 集計処理（SUM）はDB側で実施
* GitHub APIは逐次処理（負荷抑制）

---

## ■ 日付処理

* month（YYYY-MM）はServiceでLocalDateに変換
* 月末日はYearMonthで算出

---

## ■ ソート方針

| API  | ソート        |
| ---- | ---------- |
| 目標一覧 | month DESC |
| CSV  | date ASC   |

---

## ■ 外部API

| API        | 用途      |
| ---------- | ------- |
| GitHub API | コミット数取得 |
