# StudyTracker 基本設計書

---

## 1. システム全体像

StudyTrackerは、ユーザーや外部ツールからのリクエストを受け取り、学習記録の管理や分析を行うAPIサービスとして提供されます。

* Swagger UIまたはPostmanからStudyTracker APIにリクエストを送信する
* StudyTracker APIはコントローラでリクエストを受け付け、必要に応じてデータベースや外部APIと連携する
* データはH2 Databaseに保存する
* GitHub連携により、GitHub APIにリクエストを送信してコミット情報を取得する


```
+---------------------+           +----------------------+           +---------------------+
|                     |           |                      |           |                     |
|  Swagger UI /       |  HTTP     |  StudyTracker API    |  REST     |  GitHub API         |
|  Postman            +---------->+                      +---------->+  (コミット情報取得) |
|                     |           |                      |           |                     |
+---------------------+           +---------+------------+           +---------------------+
                                            |
                                            | CRUD / 認証
                                            v
                                   +---------------------+
                                   |                     |
                                   | H2 Database         |
                                   | (学習記録 / タグ /  |
                                   | 目標など)           |
                                   |                     |
                                   +---------------------+
```

## 2. 機能構成（機能一覧）

システムの機能は大きく以下のカテゴリに分類されます。

### ① ユーザー系

ユーザーの識別や外部サービスとの接続に関する機能。

* 認証機能（ユーザー登録・ログイン）

### ② 学習記録コア

アプリの中心となる学習データの管理機能。

* 学習記録管理

  * 日付、科目、学習時間（分）、メモ、タグを記録
* タグ管理
* メモ検索

### ③ 可視化・分析系

学習記録を見える化・分析する機能。

* 学習カレンダー（日ごとの学習時間表示）
* 学習統計（日別／月別／科目別の学習時間集計）
* GitHub連携（コミット数取得）
* ストリーク表示（連続学習日数）

### ④ 目標・モチベーション管理

学習の継続や達成を支援する機能。

* 目標管理（月単位の学習時間目標の設定・達成状況確認）

### ⑤ データ入出力・拡張

外部利用やデータの持ち出しに関する機能。

* CSVエクスポート

## 3. API単位の仕様

### 1. 認証機能

| メソッド | パス             | 概要     |
| ---- | -------------- | ------ |
| POST | /auth/register | ユーザー登録 |
| POST | /auth/login    | ログイン   |

---

### 2. ユーザー情報

| メソッド | パス        | 概要      |
| ---- | --------- | ------- |
| GET  | /users/me | 自分の情報取得 |

---

### 3. 学習記録（コア）

| メソッド   | パス                  | 概要       |
| ------ | ------------------- | -------- |
| POST   | /study-records      | 学習記録登録   |
| GET    | /study-records      | 学習記録一覧取得 |
| GET    | /study-records/{id} | 学習記録詳細   |
| PATCH  | /study-records/{id} | 学習記録更新   |
| DELETE | /study-records/{id} | 学習記録削除   |

#### 検索・フィルタ

| メソッド | パス                                           | 概要   |
| ---- | -------------------------------------------- | ---- |
| GET  | /study-records?from=YYYY-MM-DD&to=YYYY-MM-DD | 日付検索 |
| GET  | /study-records?tag=xxx                       | タグ検索 |
| GET  | /study-records?keyword=xxx                   | メモ検索 |

## 5. 認証・認可方式

* 認証方式：JWT（JSON Web Token）
* ログイン時にトークンを発行し、リクエスト時に付与して認証を行う

---

## 6. 非機能要件の具体化

* APIレスポンスの目標時間：1秒以内
* シンプルなクエリ構成とし、パフォーマンスを確保する

---

## 7. 制約・前提条件

* 使用技術：Spring Boot / H2 Database
* 個人開発を目的としたAPI提供
* 本システムはバックエンドAPIとして提供し、フロントエンドは含まない
