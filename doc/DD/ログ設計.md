# ログ設計書

## 1. 目的

本システムにおけるログは、開発時の不具合調査（デバッグ）を主目的とする。
本番運用を前提とした監査ログや分析用途は対象外とする。

---

## 2. ログ出力方針

### 2.1 出力レベル

ログレベルは以下の4種類を使用する。

| レベル   | 用途                    |
| ----- | --------------------- |
| INFO  | 処理の開始・終了などの主要イベント     |
| DEBUG | リクエストパラメータや内部値などの詳細情報 |
| WARN  | 想定外だが処理継続可能な事象        |
| ERROR | 例外や処理失敗               |

---

### 2.2 出力対象

ログ出力対象は以下とする。

* Controller
* Service
* ExceptionHandler
* Security（既存処理を流用）

Repository層およびEntity層ではログ出力を行わない。

---

### 2.3 出力内容

#### 出力する情報

* メソッド開始・終了
* リクエストパラメータ（Controller）
* 処理結果（件数、IDなど）
* エラー内容

#### 出力しない情報

* パスワード
* JWTトークン全文
* その他の機密情報

---

### 2.4 ログフォーマット

シンプルなテキスト形式とする。

例：

```
[UserService] createUser start
[UserService] createUser end: userId=1
[AuthController] login request: username=xxx
```

---

## 3. 実装方式

* ログライブラリ：slf4j + logback（Spring Boot標準）
* Lombokの @Slf4j を使用してログ出力を行う

---

## 4. AOPによるログ出力

### 4.1 方針

AOPを使用して、以下のログを自動出力する。

* メソッド開始（INFO）
* メソッド終了（INFO）
* 例外発生時（ERROR）

---

### 4.2 対象

* Controller
* Service

---

### 4.3 注意事項

* AOPでは詳細ログ（パラメータ・結果）は出力しない
* 詳細情報は各クラス内で個別にログ出力する
* 二重ログを防ぐため、役割を明確に分離する

---

## 5. 各層のログルール

### 5.1 Controller

* リクエスト受信時にDEBUGログを出力
* パラメータを含める

```java
log.debug("[UserController] createUser request: username={}", request.getUsername());
```

---

### 5.2 Service

* 必要に応じて処理結果をログ出力

```java
log.debug("[UserService] createUser result: userId={}", user.getId());
```

---

### 5.3 ExceptionHandler

* 例外発生時にERRORログを出力

```java
log.error("[GlobalExceptionHandler] error: {}", ex.getMessage(), ex);
```

---

### 5.4 Security

* 既存ログをベースに、フォーマットを統一する

---

## 6. 設定（application.yml）

```yaml
logging:
  level:
    root: INFO
    com.example.studytracker: DEBUG
```
