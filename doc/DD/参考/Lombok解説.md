以下は Project Lombok でよく使われるアノテーションの簡単な説明です：

---

### ■ `@Getter`

* 各フィールドの **getterメソッド（取得用メソッド）を自動生成**
* 例：`private String name;` → `getName()` が作られる

---

### ■ `@NoArgsConstructor`

* **引数なしコンストラクタ（デフォルトコンストラクタ）を生成**
* フレームワーク（例：JPAなど）で必要になることが多い

---

### ■ `@AllArgsConstructor`

* **全フィールドを引数に持つコンストラクタを生成**
* すべての値を一度にセットしてインスタンス化できる

---

### ■ `@Builder`

* **ビルダーパターンを簡単に使えるようにする**
* 可読性の高いオブジェクト生成が可能
  例：

  ```java
  User user = User.builder()
      .name("Taro")
      .age(20)
      .build();
  ```

---

### まとめ

これらを使うと：
👉 ボイラープレートコード（getter・コンストラクタなど）を大幅に削減できる
👉 コードがシンプルで読みやすくなる
