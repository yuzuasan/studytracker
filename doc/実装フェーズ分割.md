# ■ 実装フェーズ分割および工数見積

本システムは段階的に開発を進めるため、機能をフェーズ単位に分割して実装する。
各フェーズごとに優先度と実装工数を定義する。

---

## ■ フェーズ構成方針

* MVP（最小構成）を最優先で構築する
* コア機能 → 補助機能 → 分析機能 → 外部連携の順で実装
* 各フェーズは単体で動作確認可能とする

---

# ■ Phase 1：MVP（コア機能）

## ■ 対象API

### 認証

* POST /auth/register
* POST /auth/login

### ユーザー

* GET /users/me

### 学習記録

* POST /study-records
* GET /study-records
* GET /study-records/{id}
* PATCH /study-records/{id}
* DELETE /study-records/{id}

---

## ■ 内容

* JWT認証の実装
* 学習記録のCRUD
* 基本的な一覧取得

---

## ■ 想定実装時間

👉 **15〜20時間**

---

## ■ 成果

* ログインして学習記録を登録・閲覧できる状態

---

# ■ 全体工数まとめ

| フェーズ    | 内容    | 工数         |
| ------- | ----- | ---------- |
| Phase 1 | MVP   | 15〜20h     |
| Phase 2 | 検索・タグ | 6〜10h      |
| Phase 3 | 分析    | 6〜10h      |
| Phase 4 | 目標    | 4〜6h       |
| Phase 5 | 拡張    | 6〜10h      |
| **合計**  |       | **40〜70h** |

---

# ■ 補足

* 最初はPhase1のみを完成させることを最優先とする
* 各フェーズ完了ごとに動作確認（Swagger UI、Postman）を実施する
* 実装中に難易度が高い箇所（JWT / 集計処理）は時間増加の可能性あり
