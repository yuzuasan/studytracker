# ■ 設計方針（重要）

---

## ■ ① 機能単位で分割

👉 controller / service / dto は機能ごと

---

## ■ ② レイヤ分離

* controller ← dto
* service ← entity
* repository ← entity

👉 依存方向を守る

---

## ■ ③ DTOは細かく分割

👉 Request / Response分離

---

## ■ ④ securityは独立

👉 他と混ぜない
